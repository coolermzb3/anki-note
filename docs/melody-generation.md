# 自动旋律生成规则

自动旋律生成使用版本化算法 `melody-v2`。目标是在所选音域内兼顾两件事：每个八音乐句保持局部旋律性，并在一次练习中逐步覆盖较宽的音区。算法只决定目标音符顺序；答题、计时、复习记录和统计方式与其他练习策略相同。

实现入口为 [`melody.ts`](../src/domain/melody.ts)，练习页通过 [`PracticeView.tsx`](../src/components/PracticeView.tsx) 为一次练习保存生成状态。确定性行为测试位于 [`melody.test.ts`](../src/domain/melody.test.ts)。

## 音高与音区

- 同一实际音高的不同谱号写法先按 `pitchId` 合并，旋律覆盖按实际音高计算；最终显示哪一种谱号写法仍从该音高的可用 `TargetNote` 中选择。
- 所有可用音高按自然音级从低到高排序。
- 音区数量为 `max(1, round(音域自然音级跨度 / 7))`，再把排序后的音高尽量均匀地分入连续音区。因此每个音区约为一个八度，实际边界会随所选音域调整。
- 当前练习分别记录每个音区的乐句访问次数和每个实际音高的出现次数。开始新练习时全部重置，不写入设置、练习记录或备份。

## 八音乐句

每个乐句固定为 8 音，前 4 音建立局部动机，后 4 音允许复用或变化动机。固定八音长度使旋律乐句继续与谱页的八音视觉分组对齐。

普通乐句音程分布如下：

| 移动 | 概率 | 规则 |
| --- | ---: | --- |
| 级进 | 65% | 随机上行或下行 1 个自然音级 |
| 中小跳 | 25% | 随机上行或下行 2 或 3 个自然音级 |
| 同音 | 10% | 保持当前音高 |

移动始终限制在当前音区内。普通移动的上下方向都可用时，候选权重为 `1 / (1 + 本次练习出现次数)`，使较少出现的具体音高更容易被选中；只有一个方向可用时直接使用该方向。动机复现越过边界时先尝试反向移动，反向也不可用时才贴近边界。

后半句对应位置有 72% 概率使用前半句动机。动机变化保留原方向与距离的概率为 65%，反向的概率为 17%，其余情况缩小音程；同音在需要变化时改为随机上下级进。

每句第 8 音优先收束到当前音区内距离最近的 C、E 或 G。生成批次以至少 2 音的非过渡残句结束时，最后一音优先收束到 C 或 G；单音批次和仍在执行音区过渡的残句保持原定移动。目标音区的起始锚点也优先从 C、E、G 中选择，并优先使用本次练习中出现次数较少的音高。

## 音区转移

每个新乐句开始时：

- 50% 概率留在当前音区。
- 50% 概率转移到其他音区；候选范围只保留乐句访问次数最少的音区，确保较宽音域不会长期集中在起始位置。
- 多个候选音区同样少用时，70% 概率选择距离当前最近的候选音区，30% 概率从所有候选音区中选择，以兼顾渐进移动和较远跳跃。
- 只有一个音区时不执行音区转移。

音区转移先确定目标音区锚点，再决定是否使用中间过渡音：

| 中间过渡音数量 | 概率 |
| ---: | ---: |
| 0 | 40% |
| 1 | 30% |
| 2 | 20% |
| 3 | 10% |

过渡音从起点与目标锚点之间所有已启用的实际音高中随机选择，保持总体移动方向但不要求等距；可用音不足时自动减少数量。过渡音和目标锚点占用新乐句的开头，且不写入四音动机。经过的音高计入音高出现次数，只有最终目标音区计为该乐句的音区访问。

过渡音始终位于起点和目标锚点之间并保持总体移动方向，因此不允许越过目标后折返，也不主动制造反向过渡。

## 参数速查

| 参数 | 当前值 |
| --- | ---: |
| 乐句长度 | 8 音 |
| 动机长度 | 4 音 |
| 音区目标宽度 | 7 个自然音级 |
| 乐句间音区转移概率 | 50% |
| 同等覆盖候选中优先最近音区 | 70% |
| 后半句采用对应动机 | 72% |
| 过渡音数量分布 | 0/1/2/3 = 40%/30%/20%/10% |

这些值是算法常量，不是用户设置。调整会改变训练序列分布；若变化会实质改变覆盖或旋律语义，应新增算法版本，而不是覆盖 `melody-v2` 的比较含义。
